V2Ray学习

学习目的： 打造个人的专属基础通信网络，理清操作流程，能够根据自己的现实要求选择合适的配置来使用网络。

特性

  1. 多入口多出口/多协议支持。一个进程同时支持多个入站协议和出站协议。
  2. 路由可定制。入站的流量可以按照配置从不同的出口发出。
  3. 多传输载体支持。每种协议都会有一至多个传输载体可供选择。
  4. 可隐蔽。部分协议提供的加密手段，可将V2Ray流量与正常网页流量混淆。
  5. 多用户。大多数入站协议都支持设置多个用户。
  6. 反向代理与多平台支持。

工作原理

  1. 在客户端，通过设置入站协议能够接受浏览器等软件的上网请求，通过设置出站协议和路由能够选择是代理连接V2Ray服务器或还是直连。
  2. 在服务端，通过设置入站协议能够选择协议和新增用户，通过设置出站协议能够过滤某些请求。

工作机制

  1. 单服务器模式。V2Ray客户端对流量进行判断，后将所需要转发的请求发送至服务器进行代理上网。支持多台设备和多个协议进行访问。
  2. 桥接模式。V2Ray客户端将全部请求发送至一台转发服务器，
    转发服务器对流量进行判断，后将所需要转发的请求发送至服务器进行代理上网。

Linux服务器安装方法

  1. 使用root权限运行
    bash <(curl -L -s https://install.direct/go.sh)
  2. 安装脚本执行完毕后，配置文件在 /etc/v2ray/config.json处。
  3. 编辑好配置文件后，大多数Linux系统均可以使用service v2ray start|stop|status|reload|restart|force-reload 控制V2Ray的运行。

常用协议详解

  1. Freedom：出站协议，能够向任意网络发送数据。

    • 使用配置实例：

    • 协议设置说明：

      • domainStrategy ：域名策略。可选值 "AsIs" (默认值)| "UseIP" | "UseIPv4" | "UseIPv6"。通常默认即可。
        • "AsIs" : 直接向此域名发出连接。
        • "UseIP": 将域名解析为 IP 之后再建立连接。
          • "UseIPv4": 将域名解析为 IPv4 地址之后再建立连接。
          • "UseIPv6": 将域名解析为 IPv6 地址之后再建立连接。
          • 解析时使用 V2Ray 内建 DNS 服务器。

      • redirect : 强制地址转发。通常默认即可。

      • 客户端使用此协议出站，将直连上网。

      • 服务器使用此协议出站，即开启代理。

    • 协议设置说明：

      • users/clients : 用户数组。每组由id、alterId、security、level四个属性组成。
        • id: VMess 用户的主 ID。是一个合法的UUID(通用唯一识别码)。
          • Linux 系统可以使用uuid程序获得。
          • 也可以使用在线网站生成UUID。
        • alterId: VMess 用户的主 ID的额外ID个数。默认为0，最大为65536。推荐值为16。
        • security: 加密方式。
          • 支持加密算法如下：
            • aes-128-gcm (推荐在电脑上使用)
            • chacha20-poly1305 (推荐在移动端使用)
          • 推荐使用默认值auto，这样服务器可以自动适配客户端选择的加密方法 。
        • level：用户等级

    • 使用此协议时，要求客户机与服务器的UTC时间相差不能超过90秒钟。

      • Linux系统可以使用命令 date -u "+%Y-%m-%dT%H:%M:%SZ" 查看当前UTC时间。

      • Windows系统可以通过运行Powershell的命令查看

常用传输方式详解

传输层，处于应用层外部，负责包装应用层协议的数据传输方式。此处的传输方式设置必须是服务器和客户端一样的，否则连接无法建立。

  1. TCP传输控制协议。这是V2Ray默认的传输方式。

  2. QUIC快速UDP网络连接。由 Google 提出的使用 UDP 进行多路并发传输的协议。示例配置如下：

    • 传输协议设定说明：

      • network指定协议名称。

      • 第一个security是传输协议设定，需设定为none，因为使用quic时服务器会自动指定一个证书开启tls。

      • 第二个security是QUIC数据包加密方式，可用选项有

        • aes-128-gcm (推荐在电脑上使用)
        • chacha20-poly1305 (推荐在移动端使用)
        • none

      • header是数据包头部伪装设置，可供type使用的值有

        • "none": 默认值
        • "srtp": 伪装成 SRTP 数据包，会被识别为视频通话数据（如 FaceTime）。
        • "utp": 伪装成 uTP 数据包，会被识别为 BT 下载数据。
        • "wechat-video": 伪装成微信视频通话的数据包。
        • "dtls": 伪装成 DTLS 1.2 数据包。
        • "wireguard": 伪装成 WireGuard 数据包。(并不是真正的 WireGuard 协议)
